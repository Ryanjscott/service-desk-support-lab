# Suspicious Email Handling Guide

## Purpose
This guide provides a basic Level 1 process for responding to suspicious email or suspected phishing reports.

## Common Indicators
- Unknown or unusual sender
- Urgent wording
- Unexpected external link
- Request for password or payment information
- Poor spelling or unusual formatting
- Unexpected attachment
- Sender address does not match the claimed organisation

## Initial Checks
1. Ask the user not to click links or download attachments.
2. Confirm whether the user clicked any links.
3. Confirm whether any attachments were opened.
4. Confirm whether credentials were entered.
5. Review the sender address and email wording.
6. Check whether the message was received by other users.
7. Report or escalate the email according to security process.

## User Guidance
Tell the user:
- Do not interact with the email.
- Do not forward it externally.
- Report it using the approved reporting process.
- Delete it only after reporting, if instructed.

## Escalation Criteria
Escalate if:
- User clicked a link
- User entered credentials
- Attachment was opened
- Multiple users received the same email
- Email appears to impersonate an internal user or supplier
- Malware or credential theft is suspected

## Notes
Record user actions clearly. The most important detail is whether the user interacted with the email.
